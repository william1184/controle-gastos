# Technical Writer Skill — User Documentation

## Objective
Create and maintain clear, user-friendly documentation for end users of the financial application.

This role transforms technical product features into accessible guides, tutorials,
FAQs, onboarding materials, and support documentation.

---

## Core Responsibilities

### User Documentation
Create documentation for:

- account setup
- onboarding
- creating budgets
- importing bank statements
- uploading invoices/receipts
- syncing with Google Drive
- syncing with OneDrive
- restoring backups
- managing categories
- reading financial reports
- using AI suggestions
- troubleshooting sync issues

---

## Documentation Types

### User Guide
Step-by-step documentation for all major features.

### Quick Start Guide
Help users start using the platform quickly.

### FAQ
Common questions and solutions.

### Troubleshooting Guide
Help users solve common issues.

### Release Notes
Explain product changes in simple language.

### Help Center Articles
Modular searchable documentation.

### In-App Help Content
Tooltips
Walkthroughs
Microcopy

---

## Writing Standards

Documentation must be:

- simple
- concise
- non-technical
- actionable
- visual when necessary
- searchable
- multilingual-ready

---

## Required Knowledge

### Product Knowledge
- personal finance workflows
- business finance workflows
- budgeting
- invoices
- OCR uploads
- financial reports

### Technical Knowledge
- Electron app behavior
- local storage limitations
- sync limitations
- backup workflows

---

## Deliverables

- onboarding guides
- feature documentation
- FAQ articles
- troubleshooting guides
- release notes
- knowledge base content

---

## Documentation Workflow

1. Understand feature behavior
2. Test feature manually
3. Identify user pain points
4. Write documentation
5. Add screenshots/videos if needed
6. Publish
7. Maintain updates
